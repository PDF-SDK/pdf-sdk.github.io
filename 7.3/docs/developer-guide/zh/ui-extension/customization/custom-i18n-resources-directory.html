# 自定义国际化词条资源

# 前提

假设在你的网站根目录下有个 asserts/ 目录，这个目录除了放其它静态资源意外，同时也用来存放国际化词条资源。在后面的文档中，我们使用 websiteRoot/assets/ 来表示这个目录。

# 配置

  1. 复制 SDK 的 lib/locals 目录到 websiteRoot/assets/.

  2. 配置 I18N 加载词条的目录

    new UIExtension.PDFUI({
    i18n: {
        absolutePath: 'websiteRoot/assets/locals'
    },
    // 已忽略其它无关参数...
});

  3. 添加不同语言的词条 在 websiteRoot/assets/locals 目录里面创建一个新目录. 这个目录名字必须是符合标准的语言标识，例如中文： zh-CN. 然后在再在语言目录中创建 ui_.json.

  4. 初始化时指定默认语言

    new UIExtension.PDFUI({
    i18n: {
        absolutePath: `websiteRoot/assets/locals`,
        lng: 'zh-CN'
    }，
    // 已忽略其他无关参数
})

# 开发环境严重

  1. 清理浏览器缓存，确保能够加载到最新的词条；
  2. 刷新浏览器，在浏览器开发这工具的 Network 面包可以看到 ui_.json 已经是指向上面行政的目录，说明配置成功！
