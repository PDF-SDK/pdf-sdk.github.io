# FoxitPDFSDK 网页版集成示例 - Vue.js

本指南提供了两个示例。 一个介绍了如何在FoxitPDFSDK for Web中快速运行Vue.js的样板示例，另一个介绍了如何将FoxitPDFSDK for Web集成到现有的Vue.js应用中。

# 快速运行Vue.js样板示例

备注:FoxitPDFSDK for Web 根目录以下简称根目录或root.

FoxitPDFSDK for Web提供了Vue.js应用的样板项目示例。 该示例可以在root/integrations/中找到。

# 前提条件

  • Nodejs and npm
  • FoxitPDFSDK for Web

# 开始

在SDK的 root/integratons/vue.js/ 下，开启命令行终端，执行以下命令来初始化项目：

npm run setup

这个setup命令包含了两部分操作：

  • npm install - 安装依赖；
  • npm run update-sdk
    • 从SDK根目录复制 lib 到 root/integratons/vue.js/src/目录, 并自动把lib重命名为foxit-lib。
    • 复制 root/examples/license-key.js文件到root/integratons/react.js/src/目录下。

# 运行示例

运行下面命令启动开发服务，

npm run serve

现在您可以启动该应用程序了。打开浏览器，在地址栏输入<http://127.0.0.1:9103/> 来加载示例。

# 将FoxitPDFSDK for web 集成到现有的Vue.js项目中

这个集成过程假设您已经安装了@vue/cli应用，应用中所有的配置为默认。

# 前提条件

  • Nodejs and npm
  • @vue/cli
  • FoxitPDFSDK for Web

# 配置

让我们将您现有的Vue.js项目及FoxitPDFSDK for Web 的根目录简称为 VueJS 和 SDK。

  1. 在SDK根目录中，复制下列文件夹及文件到VueJS/src目录下，然后把lib重命名为foxi-lib。

    • lib文件夹。
    • ../examples/license-key.js 文件。
    • /integrations/vue.js/src/preload.js文件。

    此外，要正确引用字体库，还需要将SDK中的 external 文件夹复制到 VueJS/src/foxit-lib/assets目录下。

  2. 运行npm i -D cross-env来安装cross-env，同时在VueJS/package.json添加serve 和 build命令脚本。

    cross-env NODE_OPTIONS=--max_old_space_size=8192 NODE_ENV=development vue-cli-service serve
cross-env NODE_OPTIONS=--max_old_space_size=8192 NODE_ENV=production vue-cli-service build

    _ 这个步骤的目的是防止报内心泄露错误_

# 配置

在VueJS下添加以下两个文件的配置：

  • vue.config.js
  • .eslintignore

配置内容参考SDK项目下对应的文件。

# 组件

  • Create src/components/PDFViewer.vue in your VueJS，and reference it in src/App.vue.
  • 在VueJs项目下创建 src/components/PDFViewer.vue 组件， 在 src/App.vue中引用该组件。

组件配置内容，参考SDK 项目下对应的文件写法。

# 引用 Addons

如果你是把SDK 集成到您现有的工程中，您需要先了解一下本章节以了解如何引用插件。 我们将介绍三种引用插件的方式, 您只要根据需要选择选择其中一种就可以了。 关于插件的更多细节，请参考插件 .

# 1. 引用 Addon 碎片

碎片化引用是在7.2 及之前的版本中使用。这个方法无需任何配置，在 PDFViewer.vue 中构造 PDFUI 实例时按如下方式引用Addon：

this.pdfui = new UIExtension.PDFUI({
    addons: [
        the_path_to_foxit_lib + '/uix-addons/file-property/addon.info.json',
        the_path_to_foxit_lib + '/uix-addons/full-screen/addon.info.json',
        // .etc
    ],
    // other options
});

其中，the_path_to_foxit_lib 浏览器访问SDK lib目录的地址。

# 2. 模块化导入 Addon

  1. Install

    npm i -D @foxitsoftware/addon-loader

  2. 修改vue.config.js， 参考 /integrations/vue.js/vue.config.js

  3. 在PDFViewer.vue中导入各个 addon 的 addon.info.json

        import * as UIExtension from '../foxit-lib/UIExtension.full.js'
    import filePropertyAddon from '../foxit-lib/uix-addons/file-property/addon.info.json';
    import multiMediaAddon from '../foxit-lib/uix-addons/multi-media/addon.info.json';
    import passwordProtectAddon from '../foxit-lib/uix-addons/password-protect/addon.info.json';
    import redactionAddon from '../foxit-lib/uix-addons/redaction/addon.info.json';
    import pathObjectsAddon from '../foxit-lib/uix-addons/path-objects/addon.info.json';
    import printAddon from '../foxit-lib/uix-addons/print/addon.info.json';
    import fullScreenAddon from '../foxit-lib/uix-addons/full-screen/addon.info.json';
    import importFormAddon from '../foxit-lib/uix-addons/import-form/addon.info.json';
    import exportFormAddon from '../foxit-lib/uix-addons/export-form/addon.info.json';
    import undoRedoAddon from '../foxit-lib/uix-addons/undo-redo/addon.info.json';
    import textObjectAddon from '../foxit-lib/uix-addons/text-object/addon.info.json';

    同时构造PDFUI对象是这样传参：

    const libPath = '/foxit-lib/';
    this.pdfui = new UIExtension.PDFUI({
        addons: [
            filePropertyAddon,
            multiMediaAddon,
            passwordProtectAddon,
            redactionAddon,
            pathObjectsAddon,
            printAddon,
            fullScreenAddon,
            importFormAddon,
            exportFormAddon,
            undoRedoAddon
        ].concat(
            // 移动端不支持text-object addon
            UIExtension.PDFViewCtrl.DeviceInfo.isMobile
                ? []
                : textObjectAddon
        ),
        // other options
    });

# 3. 引用 allInOne.js

在foxit-lib/uix-addons/目录下有一个已经合并好的allInOne.js，在PDFViewer.vue中这样导入：

// ...
import * as UIExtension from '../foxit-lib/UIExtension.full.js';
import * as Addons from '../foxit-lib/uix-addons/allInOne.js';
// ...

在构造 PDFUI 实例时这样传参给PDFUI:

this.pdfui = new UIExtension.PDFUI({
    addons: UIExtension.PDFViewCtrl.DeviceInfo.isMobile
        ? Addons.filter(it => it.getName() !== 'textEditObject')
        : Addons,
    // other options
});

# 三种引用方式对比

引用方式 配置 增加的浏览器请求 可修改(国际化词条,addon.info.json中的配置==)
碎片化 无 n+ 支持修改
模块化 要配置gulp 0 支持修改，修改后需重新合并
allInOne 无 一个 不支持修改

# 运行您的应用程序

在命令行终端，运行：

npm run serve

太棒了，一切准备就绪。在浏览器中，输入您以上配置的地址来启动您的应用程序。
